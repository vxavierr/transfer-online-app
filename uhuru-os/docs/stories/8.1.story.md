# Story 8.1 — Agent Harness — Engine and Tool Registry

**Epic:** 8 — AI Agent (Harness, Tools, Chat)
**Status:** Draft
**Points:** 13

## Description

As a developer, I want the agent harness engine with tool registry, context builder, and session management, so that the AI agent can process conversations and call tools.

## Acceptance Criteria

1. Engine implementa o loop conversacional completo: recebe mensagem → carrega/cria sessao → verifica context window (compactacao se > 80k tokens) → monta system prompt → chama Claude API com streaming → processa `tool_use` blocks → executa tools via registry → adiciona resultados ao historico → loop ate resposta textual; streaming via SSE para o cliente
2. `ToolRegistry` implementado: metodos `register(tool)`, `getEnabled(role)` (filtra por `permissions.requiredRole`), `getSchemas(role)` (retorna formato `ClaudeTool[]` para a API), `execute(name, input, ctx)` (valida input com Zod, chama `tool.call`)
3. Context builder monta system prompt hierarquico: instrucoes da plataforma, info do usuario (nome, role), tools disponiveis, estado atual do workspace (clientes ativos, alertas nao-acknowledged)
4. Session management: `SessionStore` com `get(sessionId)`, `create(userId)`, `update(session)` usando tabela `agent_sessions`; mensagens armazenadas como jsonb; `token_count` atualizado apos cada round
5. `POST /api/v1/agent/chat` aceita `{ message, session_id? }` e retorna `text/event-stream` SSE com eventos: `text_delta`, `tool_use_start`, `tool_use_result`, `confirmation_required`, `done`
6. Testes unitarios (Vitest) com cobertura >= 80% para: `engine/conversation.ts`, `tools/registry.ts`, `context/system-prompt.ts`, `session/store.ts`

## Technical Notes

### UhuruTool Interface
```typescript
// packages/agent-harness/src/tools/types.ts
export interface UhuruTool {
  name: string                    // snake_case, ex: list_clients
  description: string             // Descricao para o Claude
  inputSchema: ZodSchema          // Validacao + geracao de JSON schema
  permissions: {
    requiredRole: 'member' | 'owner'
    isReadOnly: boolean
    isDestructive: boolean        // true = requer confirmacao
  }
  isEnabled(ctx: ToolContext): boolean
  call(input: unknown, ctx: ToolContext): Promise<ToolResult>
}

export interface ToolContext {
  userId: string
  userRole: 'owner' | 'member'
  supabase: SupabaseClient<Database>
  services: ServiceRegistry       // Acesso aos services do Hono
}
```

### SSE Events
```typescript
// Formato: data: {JSON}\n\n
type SSEEvent =
  | { type: 'text_delta'; content: string }
  | { type: 'tool_use_start'; tool: string; input: unknown }
  | { type: 'tool_use_result'; tool: string; result: unknown }
  | { type: 'confirmation_required'; tool: string; input: unknown }
  | { type: 'done'; session_id: string; usage: { input_tokens: number; output_tokens: number } }
  | { type: 'error'; message: string }
```
