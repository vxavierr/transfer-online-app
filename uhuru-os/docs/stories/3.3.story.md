# Story 3.3 — Campaign Alerts — Threshold-Based Notifications

**Epic:** 3 — Campaigns Integration (Meta + Google Ads)
**Status:** Draft
**Points:** 5

## Description

As a user, I want automatic alerts when campaign metrics cross defined thresholds (e.g., ROAS below target, budget exhausting), so that I can react quickly to performance issues.

## Acceptance Criteria

1. Tabela `alert_rules` criada via migration: `id` (uuid PK), `client_id` (uuid NOT NULL FK clients(id) ON DELETE CASCADE), `metric` (text NOT NULL CHECK IN ('roas','spend','ctr','cpc')), `operator` (text NOT NULL CHECK IN ('below','above')), `threshold` (numeric NOT NULL), `is_active` (boolean NOT NULL DEFAULT true), `created_by` (uuid NOT NULL FK profiles(id)), `created_at` (timestamptz)
2. Tabela `alerts` criada: `id` (uuid PK), `alert_rule_id` (uuid NOT NULL FK alert_rules(id) ON DELETE CASCADE), `campaign_id` (text NOT NULL), `current_value` (numeric NOT NULL), `triggered_at` (timestamptz NOT NULL DEFAULT now()), `acknowledged_at` (timestamptz NULLABLE), `acknowledged_by` (uuid NULLABLE FK profiles(id)); indice `idx_alerts_unacknowledged` WHERE `acknowledged_at IS NULL`
3. API CRUD para `alert_rules`: `GET /api/v1/clients/:id/alert-rules`, `POST /api/v1/clients/:id/alert-rules`, `PUT /api/v1/clients/:id/alert-rules/:ruleId`, `DELETE /api/v1/clients/:id/alert-rules/:ruleId`
4. API para `alerts`: `GET /api/v1/alerts` (com filtros `?client_id=&acknowledged=false`), `POST /api/v1/alerts/:id/acknowledge`, `GET /api/v1/alerts/count` (para badge no topbar)
5. Verificacao de thresholds executada a cada refresh de metricas (quando dados sao salvos no `campaign_metrics_cache`); se condicao atingida, insere novo `alert` (evitando duplicatas para a mesma regra+campanha no mesmo dia)
6. UI: badge de alertas nao-acknowledged no topbar com contagem; pagina ou dropdown de alertas com lista, cliente, metrica, valor atual vs threshold, botao "Acknowledge"

## Technical Notes

### Verificacao de Threshold
```typescript
// Executada pelo metrics-cache.service.ts apos cada upsert
async function checkAlerts(clientId: string, platform: string, metrics: CampaignMetrics[]) {
  const rules = await supabase.from('alert_rules')
    .select('*').eq('client_id', clientId).eq('is_active', true)

  for (const metric of metrics) {
    for (const rule of rules) {
      const value = metric[rule.metric]  // ex: metric.roas
      const triggered =
        (rule.operator === 'below' && value < rule.threshold) ||
        (rule.operator === 'above' && value > rule.threshold)

      if (triggered) {
        // Evitar duplicata: verificar se ja existe alert para esta regra+campanha hoje
        const today = new Date().toISOString().split('T')[0]
        const existing = await supabase.from('alerts')
          .select('id').eq('alert_rule_id', rule.id).eq('campaign_id', metric.campaign_id)
          .gte('triggered_at', today).single()
        if (!existing.data) {
          await supabase.from('alerts').insert({ alert_rule_id: rule.id, campaign_id: metric.campaign_id, current_value: value })
        }
      }
    }
  }
}
```
