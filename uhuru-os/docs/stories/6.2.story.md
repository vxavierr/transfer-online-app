# Story 6.2 — Automation Triggers

**Epic:** 6 — Automations and n8n Integration
**Status:** Draft
**Points:** 5

## Description

As a user, I want to create triggers that automatically execute workflows when conditions are met (e.g., ROAS drops below threshold), so that the platform reacts automatically to performance changes.

## Acceptance Criteria

1. Tabela `automation_triggers` criada via migration: `id` (uuid PK), `name` (text NOT NULL), `client_id` (uuid NULLABLE FK clients(id) ON DELETE CASCADE — null = todos os clientes), `condition_metric` (text NOT NULL CHECK IN ('roas','spend','ctr','cpc')), `condition_operator` (text NOT NULL CHECK IN ('below','above')), `condition_value` (numeric NOT NULL), `workflow_id` (text NOT NULL — n8n workflow ID), `is_active` (boolean NOT NULL DEFAULT true), `last_triggered_at` (timestamptz NULLABLE), `created_by` (uuid NOT NULL FK profiles(id)), `created_at` (timestamptz)
2. Tabela `automation_trigger_logs` criada: `id` (uuid PK), `trigger_id` (uuid NOT NULL FK automation_triggers(id) ON DELETE CASCADE), `campaign_id` (text NOT NULL), `metric_value` (numeric NOT NULL), `workflow_execution_id` (text NULLABLE), `status` (text NOT NULL CHECK IN ('success','failed')), `error_message` (text NULLABLE), `created_at` (timestamptz)
3. API CRUD de triggers: `GET /api/v1/automations/triggers`, `POST /api/v1/automations/triggers`, `PUT /api/v1/automations/triggers/:id`, `DELETE /api/v1/automations/triggers/:id`, `GET /api/v1/automations/triggers/:id/logs`
4. Verificacao de condicoes executada junto com o refresh de metricas (dentro de `metrics-cache.service.ts`): para cada metrica atualizada, verifica triggers ativos; se condicao atendida, executa workflow via n8n API e registra em `automation_trigger_logs`
5. Anti-spam: um trigger nao pode ser disparado mais de uma vez por campanha nas ultimas 6 horas (verifica `automation_trigger_logs` antes de disparar)
6. UI: tab "Triggers" na pagina de Automacoes; formulario de criacao/edicao (Sheet lateral) com selects de cliente, metrica, operador, threshold e workflow; lista de triggers com status e historico de execucoes expansivel

## Scope

### IN
- Migrations `automation_triggers` e `automation_trigger_logs` com RLS (owner only para criar/editar)
- Logica de verificacao integrada ao `metrics-cache.service.ts`
- Rotas Hono de triggers
- UI: tab de triggers com form e historico

### OUT
- Triggers baseados em eventos de tarefas ou outras entidades (apenas metricas de campanha no MVP)
- Webhook inbound (n8n → Uhuru OS)
- Agendamento por horario (cron-based triggers)

## Technical Notes

### Verificacao de Triggers (integrada ao refresh de metricas)
```typescript
// apps/api/src/services/automation.service.ts
async function evaluateTriggers(metrics: CampaignMetrics[], supabase: SupabaseClient) {
  const triggers = await supabase.from('automation_triggers')
    .select('*').eq('is_active', true)

  for (const metric of metrics) {
    for (const trigger of triggers) {
      if (trigger.client_id && trigger.client_id !== metric.client_id) continue

      const value = metric[trigger.condition_metric]
      const shouldFire =
        (trigger.condition_operator === 'below' && value < trigger.condition_value) ||
        (trigger.condition_operator === 'above' && value > trigger.condition_value)

      if (!shouldFire) continue

      // Anti-spam check: 6 horas
      const sixHoursAgo = new Date(Date.now() - 6 * 60 * 60 * 1000)
      const { data: recent } = await supabase.from('automation_trigger_logs')
        .select('id').eq('trigger_id', trigger.id).eq('campaign_id', metric.campaign_id)
        .gte('created_at', sixHoursAgo.toISOString()).limit(1)

      if (recent?.length) continue

      // Executar workflow
      try {
        const exec = await n8nClient.executeWorkflow(trigger.workflow_id)
        await supabase.from('automation_trigger_logs').insert({
          trigger_id: trigger.id, campaign_id: metric.campaign_id,
          metric_value: value, workflow_execution_id: exec.id, status: 'success'
        })
      } catch (err) {
        await supabase.from('automation_trigger_logs').insert({
          trigger_id: trigger.id, campaign_id: metric.campaign_id,
          metric_value: value, status: 'failed', error_message: String(err)
        })
      }
    }
  }
}
```
