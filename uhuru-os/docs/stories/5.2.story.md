# Story 5.2 — Task Comments

**Epic:** 5 — Task Management (Kanban)
**Status:** Draft
**Points:** 3

## Description

As a user, I want to add comments to tasks, so that I can discuss and document decisions related to each task.

## Acceptance Criteria

1. Tabela `task_comments` criada via migration: `id` (uuid PK), `task_id` (uuid NOT NULL FK tasks(id) ON DELETE CASCADE), `author_id` (uuid NOT NULL FK profiles(id)), `content` (text NOT NULL), `created_at` (timestamptz NOT NULL DEFAULT now())
2. RLS em `task_comments`: quem tem acesso a tarefa tem acesso aos comentarios (SELECT segue RLS de `tasks`); INSERT para qualquer autenticado com acesso a tarefa; UPDATE/DELETE para author ou owner
3. API: `POST /api/v1/tasks/:id/comments` (criar comentario), `GET /api/v1/tasks/:id/comments` (listar cronologicamente, mais antigo primeiro)
4. UI: secao de comentarios no detalhe da tarefa (ao clicar em um card ou abrir Sheet expandida); input de texto com botao "Comentar"; lista cronologica com avatar do autor, nome, timestamp relativo e conteudo
5. Autor e timestamp exibidos em cada comentario; timestamp formatado como relativo (ex: "2 horas atras") com tooltip do datetime absoluto

## Scope

### IN
- Migration `task_comments` com RLS
- Rotas Hono de comentarios
- Secao de comentarios dentro do TaskSheet ou TaskDetailModal

### OUT
- Mencoes de usuarios (@usuario)
- Reacoes a comentarios
- Edicao de comentario (apenas delete)
- Notificacoes por email/push de novos comentarios

### RLS (task_comments)
```sql
ALTER TABLE public.task_comments ENABLE ROW LEVEL SECURITY;

-- SELECT: segue RLS da task pai
CREATE POLICY "comments_select" ON public.task_comments FOR SELECT USING (
  task_id IN (SELECT id FROM public.tasks) -- RLS de tasks aplicado implicitamente
);

-- INSERT: autenticado com acesso à task
CREATE POLICY "comments_insert" ON public.task_comments FOR INSERT WITH CHECK (
  auth.uid() IS NOT NULL AND
  task_id IN (SELECT id FROM public.tasks)
);

-- DELETE: author ou owner
CREATE POLICY "comments_delete" ON public.task_comments FOR DELETE USING (
  author_id = auth.uid() OR public.get_user_role() = 'owner'
);
```
