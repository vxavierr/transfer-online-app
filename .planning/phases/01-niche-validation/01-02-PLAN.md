  <action>
For each of the 3 niches that survived the Go/No-go gate in Plan 01, create a comprehensive research document. Read 01-01-SUMMARY.md to identify the 3 survivors.

**Each research document must contain these sections:**

### 1. Market Size (IBGE/CNAE Data) — addresses NICH-02
- Identify the correct CNAE code(s) for the niche using concla.ibge.gov.br knowledge
- Use the IBGE Market Size Research Template from 01-RESEARCH.md
- Data needed: Total registered firms, firms with 5+ employees (target market), firms with 10+ employees (ideal client), top 3 states by concentration
- Revenue benchmark: average annual/monthly revenue per firm
- Capacity to pay estimate: current software spend, implied budget for new solutions
- **Source requirement:** Cite IBGE SIDRA, SEBRAE, or equivalent. If exact numbers are unavailable, provide best estimates with confidence level and source.
- **Anti-pattern:** Do NOT confuse total CNAE registrations with active target businesses. Filter by employee count.

### 2. Google Trends + Competition Analysis — addresses NICH-02
- Google Trends: Search interest for niche-related terms in Brazil (e.g., "sistema para clinica", "software juridico", "CRM imobiliaria")
- Trend direction: growing, stable, or declining over last 2 years
- Competition landscape: Name 2-3 existing software/service solutions targeting this niche in Brazil. What do they charge? What do they NOT solve (the gap)?
- LinkedIn presence: Estimate how many professionals/companies in this niche are active on LinkedIn Brazil

### 3. Customer Journey Map — addresses NICH-03
- Use the 3-Column Format from 01-RESEARCH.md Pattern 3
- Map at least 5 stages of the customer journey (first contact, qualification, service delivery, follow-up, billing/retention)
- For each stage: Current Process | Friction Point | AI Opportunity
- Identify 3+ friction points where the problem is MEASURABLE (time wasted, revenue lost, complaints visible)
- Be specific to the niche — generic "improve customer service" is not acceptable

### 4. Three-Gap Analysis — addresses NICH-04
- Use Pattern 4 from 01-RESEARCH.md
- Identify 2+ gaps (offer gap, result gap, or experience gap)
- For each gap: type, description, detection method used, evidence (search results, review quotes, competitor analysis)
- Evidence standard: 3+ searches with weak results for offer gap, 5+ negative reviews for result gap, pattern of complaints for experience gap

### 5. Buyer Profile
- Typical firm size (employees, monthly revenue)
- Who decides to buy (owner alone, manager, committee)
- How they currently buy software/services (referral, search, sales rep)
- Typical monthly spend on operational tools
- Budget cycle (monthly vs annual commitment preference)

**Research approach:** Use EXA MCP for web searches if available. Otherwise use WebFetch for specific URLs (IBGE SIDRA, SEBRAE reports, Google Trends). Supplement with domain knowledge where hard data is unavailable — always mark confidence level (HIGH/MEDIUM/LOW) per data point.

**File naming:** Replace "A", "B", "C" with the actual niche identifier in the filename (e.g., research-clinicas-estetica.md). Keep a consistent naming convention across all three.

  <action>
Using data from the 3 research documents, create the 2x2 matrix comparison per user's locked decision.

**Matrix structure** (from 01-RESEARCH.md Pattern 2):

**X-axis: Revenue/Growth Potential (1-5 scale)**
Score = (market size points) + (avg ticket points) + (replication points)
- Market size: >100k firms = 5, 50-100k = 4, 20-50k = 3, 10-20k = 2, <10k = 1
- Avg ticket potential: R$38k setup possible = 5, R$20-38k = 4, R$10-20k = 3, R$5-10k = 2
- Replication ease: Same playbook works for 100s = 5, needs heavy customization = 1

**Y-axis: Ease of Entry (1-5 scale)**
Score = (network access) + (competition density) + (decision-maker approachability)
- Network access: User has 5+ contacts = 5, 2-4 = 3, 0-1 = 1
  NOTE: User's personal network coverage per niche is UNKNOWN. Score conservatively (2-3) unless the user provided info in Plan 01 checkpoint. Flag this as an open question for user input.
- Competition: Few AI-First competitors = 5, many = 1
- Decision-maker: Owner decides alone = 5, committee needed = 1

**Document structure:**
1. Scoring detail per niche (show the math for each sub-score)
2. ASCII visual matrix placing each niche in a quadrant
3. Quadrant assignment per niche (Ideal Target, Hard Win, Quick Start, Avoid)
4. Comparative strengths/weaknesses table (what each niche does better/worse than others)
5. **RECOMMENDATION:** Name the top-right quadrant winner with 3 reasons why
6. **RISK NOTES:** What could go wrong with the recommendation (honest assessment)
7. **OPEN QUESTION FOR USER:** "How many contacts do you have in each of these 3 niches?" — the answer may change the Ease of Entry scores significantly

This document is the primary input for the user's decision on which niche to validate in Plan 03.
