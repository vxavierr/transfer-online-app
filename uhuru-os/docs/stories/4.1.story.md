# Story 4.1 — General Dashboard — Aggregated Metrics and Client Cards

**Epic:** 4 — Dashboards (General + Per-Client)
**Status:** Draft
**Points:** 8

## Description

As a user, I want a general dashboard showing aggregated metrics across all clients and per-client status cards, so that I can see the overall health of the operation at a glance.

## Acceptance Criteria

1. Metricas agregadas no topo: total spend do periodo, ROAS medio ponderado (spend-weighted), total de clientes ativos, total de alertas nao-acknowledged
2. Cards por cliente: nome, status badge, spend do periodo, ROAS, numero de campanhas ativas, numero de alertas ativos; ordenados por total spend decrescente
3. Filtro de periodo funcional (7d, 30d, 90d, custom com date range picker); filtro persistido na URL como query params
4. Cards clicaveis que navegam para `/clients/:id` (Dashboard por Cliente)
5. `GET /api/v1/campaigns/aggregate?period=30d` retorna metricas agregadas por cliente e totais globais; dados consumidos do `campaign_metrics_cache`
6. Loading skeletons durante carregamento; empty state quando nenhum cliente ativo ou sem dados de campanhas

## Scope

### IN
- Endpoint `GET /api/v1/campaigns/aggregate` com suporte a filtros de periodo
- Componentes: `DashboardMetricCard`, `ClientCard`, `PeriodFilter`
- Pagina `DashboardPage` implementada (substitui placeholder da Story 1.3)
- Supabase Realtime subscription para atualizar contagem de alertas sem refresh

### OUT
- Graficos de historico (Story 4.2)
- Drill-down detalhado por cliente (Story 4.2)
- Notificacoes em tempo real de novas metricas (requer background job — pos-MVP)

## Technical Notes

### Endpoint Aggregate
```typescript
// GET /api/v1/campaigns/aggregate
// Query: period (7d | 30d | 90d | custom com from/to)
// Logica: busca em campaign_metrics_cache para o periodo
// Agrega por cliente: SUM(spend), weighted avg ROAS, COUNT(distinct campaigns)
// Totais globais: SUM de todos os clientes
// RLS: automatico — so retorna clientes visíveis ao usuario logado
```

### Calculo ROAS Ponderado
```typescript
// ROAS medio ponderado por spend
const weightedRoas = clients.reduce((acc, c) => acc + c.roas * c.spend, 0) /
  clients.reduce((acc, c) => acc + c.spend, 0)
```

### Supabase Realtime (badge de alertas)
```typescript
// apps/web/src/hooks/useAlertCount.ts
supabase.channel('alerts').on('postgres_changes', {
  event: 'INSERT', schema: 'public', table: 'alerts'
}, () => refetchAlertCount()).subscribe()
```

### UI Layout
```
DashboardPage
├── PeriodFilter (7d | 30d | 90d | Custom)
├── GlobalMetrics (4 MetricCards: Total Spend, ROAS Medio, Clientes Ativos, Alertas)
└── ClientCards Grid (responsive: 1 col mobile, 2 col tablet, 3 col desktop)
    └── ClientCard (nome, status, spend, roas, campanhas, alertas)
```

### Performance
- Usar `react-query` com `staleTime: 5 * 60 * 1000` (5min) para evitar refetch excessivo
- Filtro de periodo muda `queryKey` para invalidar cache corretamente
