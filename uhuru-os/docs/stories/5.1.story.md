# Story 5.1 — Task Module — CRUD and Kanban Board

**Epic:** 5 — Task Management (Kanban)
**Status:** Draft
**Points:** 8

## Description

As a user, I want a Kanban board to manage tasks with drag-and-drop status changes, so that I can organize and track operational work.

## Acceptance Criteria

1. Tabela `tasks` criada via migration: `id` (uuid PK), `title` (text NOT NULL), `description` (text NULLABLE), `client_id` (uuid NULLABLE FK clients(id) ON DELETE SET NULL), `assigned_to` (uuid NULLABLE FK profiles(id) ON DELETE SET NULL), `status` (text NOT NULL CHECK IN ('backlog','in_progress','review','done') DEFAULT 'backlog'), `priority` (text NOT NULL CHECK IN ('low','medium','high','urgent') DEFAULT 'medium'), `due_date` (date NULLABLE), `created_by` (uuid NOT NULL FK profiles(id)), `created_at` (timestamptz), `updated_at` (timestamptz); indices: `idx_tasks_status`, `idx_tasks_client`, `idx_tasks_assigned`
2. RLS em `tasks`: owner ve todas; member ve tarefas que criou (`created_by = auth.uid()`) ou que lhe foram atribuidas (`assigned_to = auth.uid()`)
3. API Hono com validacao Zod: `GET /api/v1/tasks` (lista com filtros: `?client_id=&assigned_to=&status=&priority=`), `POST /api/v1/tasks`, `GET /api/v1/tasks/:id`, `PUT /api/v1/tasks/:id`, `DELETE /api/v1/tasks/:id`
4. UI: Kanban board com 4 colunas (Backlog, Em andamento, Review, Concluido); cada coluna exibe cards das tarefas com titulo, cliente, prioridade badge, responsavel avatar, data de vencimento
5. Drag-and-drop entre colunas (usando `@dnd-kit/core`) atualiza o campo `status` via `PUT /api/v1/tasks/:id` imediatamente
6. Formulario de criacao/edicao em Sheet lateral com todos os campos e validacao; cliente e responsavel sao selects com busca
7. Filtros acima do Kanban: por cliente, responsavel, prioridade; filtros persistidos na URL

  client_id: z.string().uuid().nullable().optional(),
