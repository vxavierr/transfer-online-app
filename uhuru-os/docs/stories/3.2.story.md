# Story 3.2 — Google Ads API — OAuth Flow and Campaign Listing

**Epic:** 3 — Campaigns Integration (Meta + Google Ads)
**Status:** Draft
**Points:** 8

## Description

As a user, I want to connect my Google Ads account via OAuth and see my campaigns with real metrics, so that I can monitor Google ad performance from Uhuru OS.

## Acceptance Criteria

1. Fluxo OAuth 2.0 completo para Google Ads: botao "Conectar Google Ads" na tela de detalhe do cliente inicia redirect OAuth; callback salva access_token e refresh_token encriptados em `ad_platform_tokens`
2. Refresh automatico do access_token quando expirado (Google access tokens expiram em 1h): service layer detecta expiracao e usa refresh_token antes de chamar a API
3. `GET /api/v1/clients/:id/campaigns?platform=google` retorna campanhas com metricas identicas ao formato Meta: spend, impressions, clicks, ctr, cpc, roas, status
4. Service layer chama Google Ads API (via `google-ads-api` npm ou REST direto) com os tokens do cliente
5. Cache de metricas compartilhado com Meta (tabela `campaign_metrics_cache`, TTL 30min), plataforma diferenciada pelo campo `platform='google'`
6. UI: campanhas Google aparecem na mesma tela de campanhas com filtro `platform=google`; indicador visual de plataforma (logo Meta vs Google)

## Scope

### IN
- Rotas OAuth: `GET /api/v1/auth/google-ads/authorize`, `GET /api/v1/auth/google-ads/callback`
- Google Ads API client em `apps/api/src/lib/google-ads.ts`
- Logica de refresh token automatico
- Extensao do `campaign.service.ts` para suportar Google
- UI: botao OAuth na tela de detalhe do cliente, indicador de status de conexao

### OUT
- Google OAuth para login de usuarios (diferente de Google Ads OAuth — este e para vincular conta de ads)
- Google Sheets (Story 7.2)
- Alertas (Story 3.3)

## Technical Notes

### OAuth Flow
```
1. Frontend: usuario clica "Conectar Google Ads" para cliente X
2. Frontend: redirect para GET /api/v1/auth/google-ads/authorize?client_id={uuid}
3. API: redireciona para Google OAuth com scope google-ads.readonly e state={client_id}
4. Google: usuario autoriza → redirect para /api/v1/auth/google-ads/callback?code=&state=
5. API: troca code por tokens, salva em ad_platform_tokens, redireciona para /clients/{id}
```

### Google Ads API — Metricas
```typescript
// Usando Google Ads Query Language (GAQL)
const query = `
  SELECT campaign.id, campaign.name, campaign.status,
    metrics.cost_micros, metrics.impressions, metrics.clicks,
    metrics.ctr, metrics.average_cpc, metrics.conversions_value, metrics.conversions
  FROM campaign
  WHERE segments.date DURING LAST_30_DAYS
`
// cost_micros / 1_000_000 = spend em BRL/USD
// roas = conversions_value / (cost_micros / 1_000_000)
```

### Refresh Token Logic
```typescript
async function getValidToken(clientId: string): Promise<string> {
  const tokenRecord = await getToken(clientId, 'google')
  if (tokenRecord.expires_at && tokenRecord.expires_at < new Date()) {
    const refreshed = await refreshGoogleToken(tokenRecord.refresh_token)
    await updateToken(clientId, 'google', refreshed)
    return refreshed.access_token
  }
  return tokenRecord.access_token
}
```
