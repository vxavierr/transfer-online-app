# Story 2.2 — Client Module — CRUD and Ads Account Linking

**Epic:** 2 — Client Management and Base UI
**Status:** Draft
**Points:** 8

## Description

As a user, I want to create, view, edit, and delete clients with their linked ads accounts, so that I can manage my client portfolio.

## Acceptance Criteria

1. Tabela `clients` criada via migration: `id` (uuid PK), `name` (text NOT NULL), `cnpj` (text NULLABLE), `contact_email` (text NULLABLE), `contact_phone` (text NULLABLE), `status` (text NOT NULL CHECK IN ('active','paused','ended') DEFAULT 'active'), `meta_account_id` (text NULLABLE), `google_account_id` (text NULLABLE), `created_by` (uuid NOT NULL FK profiles(id)), `created_at` (timestamptz), `updated_at` (timestamptz); indices: `idx_clients_status`, `idx_clients_created_by`
2. RLS em `clients`: owner ve todos; member ve clientes que criou (`created_by = auth.uid()`) ou que tem tarefas atribuidas a si
3. API Hono com validacao Zod: `GET /api/v1/clients` (lista com filtros `?status=&search=`), `POST /api/v1/clients`, `GET /api/v1/clients/:id`, `PUT /api/v1/clients/:id`, `DELETE /api/v1/clients/:id` (owner only)
4. Tela de listagem com tabela (colunas: nome, CNPJ, status, contas vinculadas, acoes), busca por nome em tempo real, filtro por status
5. Formulario de criacao/edicao em Sheet lateral (nao pagina separada) com todos os campos e validacao frontend via react-hook-form + zod
6. Confirmacao antes de deletar cliente (Dialog de confirmacao com nome do cliente)
7. Toast de sucesso/erro (via Sonner) em todas as operacoes CRUD
8. Response da API segue formato padrao: `{ data: Client | Client[], meta?: Pagination }` / `{ error: { code, message } }`

### RLS Policies (clients)
```sql
ALTER TABLE public.clients ENABLE ROW LEVEL SECURITY;

-- SELECT: owner vê todos; member vê criados por si ou com tasks atribuídas
CREATE POLICY "clients_select" ON public.clients FOR SELECT USING (
  public.get_user_role() = 'owner'
  OR created_by = auth.uid()
  OR id IN (SELECT client_id FROM public.tasks WHERE assigned_to = auth.uid())
);

CREATE POLICY "clients_insert" ON public.clients FOR INSERT WITH CHECK (auth.uid() IS NOT NULL);
CREATE POLICY "clients_update" ON public.clients FOR UPDATE USING (
  public.get_user_role() = 'owner' OR created_by = auth.uid()
);
CREATE POLICY "clients_delete" ON public.clients FOR DELETE USING (public.get_user_role() = 'owner');
```
