# Story 2.3 — Client Documents — File Upload via Supabase Storage

**Epic:** 2 — Client Management and Base UI
**Status:** Draft
**Points:** 5

## Description

As a user, I want to upload and manage documents (contracts, media plans) for each client, so that all client files are centralized.

## Acceptance Criteria

1. Tabela `client_documents` criada via migration: `id` (uuid PK), `client_id` (uuid NOT NULL FK clients(id) ON DELETE CASCADE), `file_name` (text NOT NULL), `file_path` (text NOT NULL — path no Storage), `file_type` (text NOT NULL — MIME type), `file_size` (bigint NOT NULL — bytes), `uploaded_by` (uuid NOT NULL FK profiles(id)), `created_at` (timestamptz NOT NULL DEFAULT now())
2. RLS em `client_documents`: mesmas permissoes da tabela `clients` pai (quem ve o cliente ve os documentos); delete permite `owner` ou `uploaded_by = auth.uid()`
3. Bucket Supabase Storage `client-documents` criado com politica de acesso: upload autenticado; download/delete seguindo RLS de `client_documents`
4. API: `POST /api/v1/clients/:id/documents` (upload multipart, max 50MB), `GET /api/v1/clients/:id/documents` (lista), `DELETE /api/v1/clients/:id/documents/:docId`, `GET /api/v1/clients/:id/documents/:docId/download` (retorna signed URL valida por 1 hora)
5. UI: secao "Documentos" na tela de detalhe do cliente (tab ou secao), com upload drag-and-drop, lista de arquivos (nome, tamanho, data, tipo, botoes de download e delete)
6. Tipos de arquivo aceitos: PDF, DOC, DOCX, XLS, XLSX, CSV, PNG, JPG (validado no backend)

## Scope

### IN
- Migration `client_documents`
- RLS e bucket Storage
- Service layer `client-documents.service.ts`
- Rotas Hono para documentos
- Componente de upload drag-and-drop e lista de documentos na tela de detalhe do cliente

### OUT
- Preview de documentos no browser
- Versionamento de documentos
- Compartilhamento de links publicos
- Analise de documentos pelo agente (EP futura)

## Technical Notes

### Storage Path Pattern
```
client-documents/{client_id}/{uuid}-{filename}
```

### Upload API (Hono)
```typescript
// POST /api/v1/clients/:id/documents
// Content-Type: multipart/form-data
// Campos: file (File)
// 1. Validar tipo e tamanho
// 2. Upload para Supabase Storage via supabase.storage.from('client-documents').upload(path, buffer)
// 3. Inserir registro em client_documents
// 4. Retornar documento criado
```

### Download (Signed URL)
```typescript
// GET /api/v1/clients/:id/documents/:docId/download
// Gera signed URL com expiry de 3600s
const { data } = await supabase.storage
  .from('client-documents')
  .createSignedUrl(doc.file_path, 3600)
return c.json({ data: { url: data.signedUrl } })
```

### UI: ClientDetailPage (novo) ou tab em ClientsPage
- `ClientDocumentsTab.tsx` com `react-dropzone` para upload drag-and-drop
- Lista com nome, tamanho formatado (KB/MB), data de upload, icone por tipo
- Click em "Download" abre signed URL em nova aba
- Delete com confirmacao (Dialog)
