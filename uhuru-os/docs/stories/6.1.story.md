# Story 6.1 — n8n Workflow Management Interface

**Epic:** 6 — Automations and n8n Integration
**Status:** Draft
**Points:** 5

## Description

As a user, I want to list, activate, pause, and execute n8n workflows from Uhuru OS, so that I can manage automations without accessing n8n directly.

## Acceptance Criteria

1. Configuracao do endpoint n8n e API key armazenados de forma segura (variaveis de ambiente `N8N_BASE_URL` e `N8N_API_KEY` no servidor Hono — nunca expostos ao frontend)
2. API proxy: `GET /api/v1/automations/workflows` lista workflows do n8n; `POST /api/v1/automations/workflows/:id/activate` ativa; `POST /api/v1/automations/workflows/:id/deactivate` desativa; `POST /api/v1/automations/workflows/:id/execute` executa manualmente; `GET /api/v1/automations/workflows/:id/executions` lista log de execucoes
3. Service layer `automation.service.ts` chama n8n API REST com timeout de 10s e tratamento de erros (n8n retorna erro 5xx → Hono retorna 502 Bad Gateway com mensagem clara)
4. UI: pagina de Automacoes com tabela de workflows (nome, status ativo/inativo, ultima execucao, tempo de execucao); botoes de acao por linha: ativar/pausar (toggle), executar manualmente
5. Confirmacao (Dialog) antes de executar workflow manualmente: exibe nome do workflow e aviso de que sera executado imediatamente
6. Estado de loading por acao individual (botao do workflow especifico exibe spinner, nao toda a tabela)

## Scope

### IN
- Service layer que encapsula a n8n API REST
- Rotas Hono proxy para workflows
- UI: lista de workflows com acoes
- Tratamento de erro quando n8n esta indisponivel (graceful degradation)

### OUT
- Triggers automaticos (Story 6.2)
- Criacao/edicao de workflows no n8n via Uhuru OS (sempre via n8n diretamente)
- Log de execucoes com detalhes de cada step (UI simplificada apenas)

## Technical Notes

### n8n API REST
```typescript
// Base: N8N_BASE_URL (ex: http://72.60.9.248:5678)
// Auth: Header 'X-N8N-API-KEY': N8N_API_KEY

// GET /api/v1/workflows → lista workflows
// PATCH /api/v1/workflows/:id → { active: true/false } para ativar/desativar
// POST /api/v1/workflows/:id/execute → executar manualmente (retorna executionId)
// GET /api/v1/executions?workflowId=:id&limit=10 → historico
```

### Client n8n
```typescript
// apps/api/src/lib/n8n.ts
export class N8nClient {
  private base = process.env.N8N_BASE_URL
  private headers = { 'X-N8N-API-KEY': process.env.N8N_API_KEY!, 'Content-Type': 'application/json' }

  async listWorkflows(): Promise<N8nWorkflow[]> {
    const res = await fetch(`${this.base}/api/v1/workflows`, { headers: this.headers, signal: AbortSignal.timeout(10_000) })
    if (!res.ok) throw new N8nError(`n8n API error: ${res.status}`)
    return (await res.json()).data
  }
  // ... outros metodos
}
```

### Autorizacao Uhuru OS
- Listar e ativar/desativar: qualquer autenticado (member pode ver, owner pode modificar)
- Executar manualmente: owner only (pode disparar automacoes)
- Implementado via role check no middleware Hono antes de proxiar para n8n
