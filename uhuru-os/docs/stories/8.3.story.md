# Story 8.3 — Agent Chat Interface and Memory Integration

**Epic:** 8 — AI Agent (Harness, Tools, Chat)
**Status:** Draft
**Points:** 13

## Description

As a user, I want a chat interface to interact with the AI agent, with persistent memory across conversations, so that the agent remembers context and provides increasingly useful assistance.

## Acceptance Criteria

1. Chat side panel acessivel de qualquer tela via botao flutuante (FAB) no canto inferior direito ou icone fixo no topbar; pode ser expandido para fullscreen
2. Interface de chat: lista de mensagens (user e assistant), input de texto com submit via Enter, botao de envio, indicador visual de "agente pensando" (dots animados)
3. Streaming: resposta do agente aparece token a token via SSE (`EventSource` ou `fetch` com ReadableStream); nao aguarda resposta completa
4. Tool calls visiveis na conversa: bloco colapsavel exibe nome da tool, input (JSON formatado) e resultado quando o agente usa uma ferramenta
5. Confirmacao de tools destrutivas: quando SSE emite `confirmation_required`, exibe prompt inline na conversa com botoes "Confirmar" e "Cancelar" — usuario pode confirmar ou cancelar antes do agente executar
6. Historico de sessoes: lista de conversas anteriores com titulo auto-gerado (primeira mensagem truncada); clicavel para carregar sessao anterior
7. Memoria de longo prazo (Mem0): apos cada resposta do agente, `post-tool hook` envia memorias relevantes ao Mem0 (VPS 72.60.9.248); no inicio de cada conversa, busca memorias relevantes via Mem0 para injetar no context builder
8. Busca semantica (Qdrant): historico de conversas indexado no Qdrant; context builder recupera chunks relevantes ao usar `memoryService.getRelevant(message, userId)`

## Scope

### IN
- Chat side panel e fullscreen mode
- Streaming SSE no frontend (EventSource ou fetch stream)
- Renderizacao de tool calls, confirmacoes
- Historico de sessoes (lista + load)
- Integracao Mem0: cliente HTTP para a API Mem0 no VPS
- Integracao Qdrant: cliente para busca vetorial
- `context/memory.ts` no agent-harness que orquestra Mem0 + Qdrant

### OUT
- Analise de documentos do cliente via agente (pos-MVP)
- Voz/audio input
- Exportacao de conversa
- Agente multimodal (apenas texto no MVP)

### Mem0 Integration
```typescript
// packages/agent-harness/src/context/memory.ts
// Mem0 endpoint: MEM0_BASE_URL (VPS)
async function getRelevant(message: string, userId: string): Promise<string[]> {
  const res = await fetch(`${MEM0_URL}/v1/memories/search`, {
    method: 'POST',
    headers: { 'Authorization': `Token ${MEM0_API_KEY}` },
    body: JSON.stringify({ query: message, user_id: userId, limit: 5 })
  })
  return (await res.json()).memories.map((m: any) => m.memory)
}

// Post-conversation: adicionar memorias ao Mem0
async function addMemory(content: string, userId: string): Promise<void> {
  await fetch(`${MEM0_URL}/v1/memories/`, {
    method: 'POST',
    headers: { 'Authorization': `Token ${MEM0_API_KEY}` },
    body: JSON.stringify({ messages: [{ role: 'user', content }], user_id: userId })
  })
}
```
