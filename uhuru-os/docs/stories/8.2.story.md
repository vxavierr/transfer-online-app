# Story 8.2 — Agent Built-in Tools — All Modules

**Epic:** 8 — AI Agent (Harness, Tools, Chat)
**Status:** Draft
**Points:** 13

## Description

As a user, I want the AI agent to have tools for all platform modules, so that I can operate the entire platform via natural language.

## Acceptance Criteria

1. 14 tools implementadas e registradas no `ToolRegistry`: `list_clients`, `get_client`, `create_client`, `update_client`, `list_campaigns`, `get_campaign_metrics`, `create_task`, `update_task`, `list_tasks`, `list_workflows`, `execute_workflow`, `create_trigger`, `import_data`, `call_api_connector`
2. Cada tool usa exclusivamente a camada de servico do Hono (nunca acessa o banco diretamente); `ctx.services` expoe acesso aos services
3. Cada tool implementa a interface `UhuruTool` com: `name` (snake_case), `description` (claro, usado pelo Claude para decidir quando usar), `inputSchema` Zod (todos os campos documentados), `permissions.requiredRole`, `permissions.isReadOnly`, `permissions.isDestructive` (true para operacoes de criacao/modificacao que nao sao revertidas facilmente)
4. Tools marcadas como `isDestructive: true`: `execute_workflow`, `create_trigger` — retornam `confirmation_required` via hook (implementado em Story 8.4)
5. Audit log: toda execucao bem-sucedida de tool grava em `agent_audit_logs` via `post-tool hook` (implementado em Story 8.4); esta story apenas garante que a interface e compativel
6. Testes unitarios para cada tool: testar `inputSchema` (rejeitar input invalido), `permissions.requiredRole` correto, e mock da camada de servico

## Scope

### IN
- 14 arquivos de tools em `packages/agent-harness/src/tools/built-in/`
- `ServiceRegistry` interface que expoe os services necessarios via `ctx.services`
- Testes unitarios por tool

### OUT
- Hooks de permissao e audit (Story 8.4)
- Chat UI (Story 8.3)
- Tool `get_client_metrics` (pode ser adicionada como extensao de `get_client`)
- Tools de gerenciamento de alertas (extensao pos-MVP)

### Exemplo de Tool
```typescript
// packages/agent-harness/src/tools/built-in/create-task.ts
import { z } from 'zod'
import type { UhuruTool, ToolContext, ToolResult } from '../types'

export const createTaskTool: UhuruTool = {
  name: 'create_task',
  description: 'Cria uma nova tarefa na plataforma Uhuru OS. Use para registrar ações a serem realizadas pela equipe.',
  inputSchema: z.object({
    title: z.string().min(1).describe('Título da tarefa'),
    description: z.string().optional().describe('Descrição detalhada'),
    client_id: z.string().uuid().optional().describe('ID do cliente vinculado (opcional)'),
    assigned_to: z.string().uuid().optional().describe('ID do usuário responsável'),
    priority: z.enum(['low','medium','high','urgent']).default('medium'),
    due_date: z.string().date().optional().describe('Data limite (YYYY-MM-DD)'),
  }),
  permissions: { requiredRole: 'member', isReadOnly: false, isDestructive: false },
  isEnabled: (_ctx) => true,
  async call(input, ctx: ToolContext): Promise<ToolResult> {
    const task = await ctx.services.task.create(input as any, ctx.userId)
    return { success: true, data: task }
  }
}
```

### Descricoes para o Claude (qualidade critica)
As descricoes devem ser especificas o suficiente para o Claude escolher a tool correta:
- `list_clients`: "Lista todos os clientes da agencia. Use para ver quais clientes existem, filtrar por status ou buscar por nome. Retorna nome, status e contas de ads vinculadas."
- `execute_workflow`: "Executa manualmente um workflow do n8n imediatamente. ATENÇÃO: esta ação é irreversível. Confirme com o usuário antes de chamar esta ferramenta."
