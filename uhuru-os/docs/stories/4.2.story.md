# Story 4.2 — Per-Client Dashboard — Detailed Metrics and Drill-Down

**Epic:** 4 — Dashboards (General + Per-Client)
**Status:** Draft
**Points:** 8

## Description

As a user, I want a detailed dashboard for each client showing campaigns, metrics history, tasks, and alerts, so that I can deeply understand each client's performance.

## Acceptance Criteria

1. Header do cliente: nome, badge de status, contas de ads vinculadas (Meta e/ou Google com indicador de conexao), botao "Editar cliente"
2. Metricas do cliente: spend, ROAS, CPA, CTR do periodo selecionado com comparacao percentual vs periodo anterior (ex: "ROAS 3.2 (+15% vs periodo anterior)")
3. Lista de campanhas ativas do cliente (Meta + Google) com metricas individuais; filtro por plataforma
4. Grafico de historico de performance: spend e ROAS ao longo do tempo para o periodo selecionado; usar `recharts` ou `tremor`
5. Secao de tarefas vinculadas ao cliente: lista das 5 mais recentes com status e prioridade; link "Ver todas" que navega para `/tasks?client_id=:id`
6. Secao de alertas ativos do cliente com opcao de acknowledge direto
7. Filtro de periodo consistente com o Dashboard Geral (7d, 30d, 90d, custom)
8. Supabase Realtime: quando novos dados chegam ao `campaign_metrics_cache` para o cliente, dashboard reflete sem refresh manual (canal por `client_id`)

## Scope

### IN
- Pagina `ClientDetailPage` com todas as secoes acima
- `GET /api/v1/clients/:id/metrics?period=30d` — metricas agregadas do cliente + comparacao
- `GET /api/v1/clients/:id/metrics/history?period=30d` — serie temporal diaria para graficos
- Componentes de grafico com `recharts`
- Realtime subscription por cliente

### OUT
- Edicao do cliente nesta pagina (abre Sheet de 2.2)
- Plano de midia ativo (pos-MVP)
- Exportacao de relatorio (pos-MVP)
- Graficos por campanha individual (pos-MVP)

### Comparacao vs Periodo Anterior
```typescript
// Se period=30d: compara com os 30 dias anteriores ao período
// Variacao: ((current - previous) / previous) * 100
// Exibir com arrow: ↑ verde se melhora, ↓ vermelho se piora (depende da metrica)
// CPA: ↓ vermelho se aumenta (custo maior = pior); ROAS: ↓ vermelho se cai
```

### Grafico (recharts)
```typescript
// LineChart com dois eixos Y: spend (esquerda, R$) e ROAS (direita)
// Tooltip com valores formatados
// Responsive via ResponsiveContainer
// Cores: spend=#6366f1, roas=#22c55e (Uhuru brand)
```

### Realtime por Cliente
```typescript
supabase.channel(`client-metrics-${clientId}`)
  .on('postgres_changes', {
    event: '*', schema: 'public', table: 'campaign_metrics_cache',
    filter: `client_id=eq.${clientId}`
  }, () => refetch())
  .subscribe()
```
