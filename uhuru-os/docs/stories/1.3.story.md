# Story 1.3 — Authentication Flow — Login, Register and Protected Routes

**Epic:** 1 — Foundation, Auth and Project Setup
**Status:** Draft
**Points:** 5

## Description

As a user, I want to register, login, and access protected routes, so that I can securely use the platform.

## Acceptance Criteria

1. Tela `/login` com campos email e password, validacao client-side, feedback de erro (credenciais invalidas) e loading state durante autenticacao
2. Tela `/register` com campos full_name, email, password e confirmacao de password; validacao de senha minima 8 caracteres e match de confirmacao
3. `AuthContext` React provider que gerencia sessao via `supabase.auth.onAuthStateChange`; expoe `user`, `session`, `loading`, `signIn`, `signOut`, `signUp`
4. Rota `/dashboard` protegida: redireciona para `/login` se sessao nao existir; exibe nome do usuario logado e resultado do `GET /health` da API
5. Middleware Hono `authMiddleware` valida JWT do Supabase em todas as rotas protegidas da API; injeta `user` e `supabaseClient` no contexto Hono
6. `GET /api/v1/auth/me` retorna perfil do usuario logado (dados da tabela `profiles`)
7. Logout funcional: limpa sessao Supabase, redireciona para `/login`
8. Tokens JWT propagados em todas as requests do frontend para a API via header `Authorization: Bearer <token>`

## Scope

### IN
- Telas de Login e Register (React)
- AuthContext/provider e hook `useAuth`
- Protected route wrapper `ProtectedRoute`
- Middleware JWT no Hono
- Rotas auth na API: `POST /api/v1/auth/register`, `POST /api/v1/auth/login`, `POST /api/v1/auth/logout`, `POST /api/v1/auth/refresh`, `GET /api/v1/auth/me`
- API client no frontend (`packages/shared` ou `apps/web/src/lib/api.ts`) que injeta JWT automaticamente

### OUT
- Layout completo da aplicacao (sidebar, topbar) — Story 2.1
- Reset de senha
- Social auth (Google OAuth para Ads e diferente de login — vem no Epic 3)
- Roles/permissoes no frontend alem de autenticado/nao-autenticado

## Technical Notes

### Hono Auth Middleware
```typescript
// apps/api/src/middleware/auth.ts
import { createMiddleware } from 'hono/factory'
import { createSupabaseClient } from '@uhuru/shared'

export const authMiddleware = createMiddleware(async (c, next) => {
  const token = c.req.header('Authorization')?.replace('Bearer ', '')
  if (!token) return c.json({ error: { code: 'UNAUTHORIZED', message: 'Missing token' } }, 401)

  const supabase = createSupabaseClient(token)
  const { data: { user }, error } = await supabase.auth.getUser(token)
  if (error || !user) return c.json({ error: { code: 'UNAUTHORIZED', message: 'Invalid token' } }, 401)

  const { data: profile } = await supabase
    .from('profiles').select('*').eq('id', user.id).single()

  c.set('user', profile)
  c.set('supabase', supabase)
  await next()
})
```
